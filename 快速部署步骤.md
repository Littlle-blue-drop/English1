## 📋 下一步操作

### 步骤1：创建GitHub仓库并推送

1. **访问GitHub并创建新仓库**
   - 打开 https://github.com/new
   - Repository name: `english-practice-app`（或您喜欢的名称）
   - 选择 Public 或 Private
   - **不要**勾选 "Initialize with README"
   - 点击 "Create repository"

2. **连接本地仓库并推送**
   
   在项目目录执行以下命令（将YOUR_USERNAME和REPO_NAME替换为实际值）：
   
   ```bash
   cd f:\cursor\20251014English2.4
   
   # 添加远程仓库
   git remote add origin https://github.com/YOUR_USERNAME/REPO_NAME.git
   
   # 重命名分支为main（如果需要）
   git branch -M main
   
   # 推送到GitHub
   git push -u origin main
   ```
   
   **示例：**
   ```bash
   git remote add origin https://github.com/john/english-practice-app.git
   git branch -M main
   git push -u origin main
   ```
   
   **⚠️ 如果遇到认证问题：**
   - GitHub已不支持密码认证
   - 需要创建Personal Access Token
   - 访问 https://github.com/settings/tokens 创建token
   - 推送时使用token作为密码

### 步骤2：在Vercel部署

1. **登录Vercel**
   - 访问 https://vercel.com
   - 点击 "Sign Up" 或 "Log In"
   - 选择 "Continue with GitHub"

2. **导入项目**
   - 点击 "Add New..." → "Project"
   - 在 "Import Git Repository" 中找到您的GitHub仓库
   - 点击 "Import"

3. **配置环境变量（⚠️ 重要！在点击Deploy之前配置）**
   
   在部署页面的 "Environment Variables" 部分，添加以下变量：
   
   ```
   NEXT_PUBLIC_XFYUN_APP_ID = 您的讯飞APP_ID
   NEXT_PUBLIC_XFYUN_API_KEY = 您的讯飞API_KEY
   NEXT_PUBLIC_XFYUN_API_SECRET = 您的讯飞API_SECRET
   NEXT_PUBLIC_SUPABASE_URL = 您的Supabase URL（例如：https://xxxxx.supabase.co）
   NEXT_PUBLIC_SUPABASE_ANON_KEY = 您的Supabase匿名密钥
   JWT_SECRET = 强随机字符串（32+字符）
   ```
   
   **生成JWT_SECRET：**
   ```bash
   node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
   ```
   
   **环境选择：** 勾选 Production、Preview、Development

4. **点击Deploy**
   - 等待2-5分钟构建完成
   - 部署成功后获得访问URL
